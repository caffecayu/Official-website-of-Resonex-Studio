## 一、官网背景层级

从最底层到最上层：

| 层级 | 实现方式 | 参数 | 作用 |
|------|----------|------|------|
| **body 背景** | CSS `radial-gradient` | `circle at top left, #183f2b 0, #080c0a 35%, #030403 100%` | 深绿→深灰→纯黑的径向渐变，左上角为光源方向 |
| **section 额外光晕** | CSS `radial-gradient` / `linear-gradient` | 各 section 独立叠加淡绿径向光晕（如 `rgba(157,255,159,0.08)`） | 给不同区块增加微弱的局部辉光 |
| **noise 层** | CSS `linear-gradient` 重复网格 | `#fff 1px` 网格，`background-size: 3px 3px`，`opacity: 0.05`，fixed 定位 | 极淡的 CRT/扫描线纹理质感 |

**混合方式**：所有背景层均为 CSS 默认叠加（覆盖），noise 在最顶层通过 `opacity: 0.05` 产生极轻微纹理。

## 二、数字地球结构（Three.js r128）

数字地球由以下层级构成，全部通过 Three.js 渲染在 `<canvas class="globe-canvas">` 上：

### 2.1 球体主体（Globe Body）

| 属性 | 值 |
|------|-----|
| 几何 | `SphereGeometry(0.995, 96, 96)` |
| 材质 | `MeshBasicMaterial` |
| 颜色 | `#0d281b`（暗墨绿） |
| 透明度 | `0.16` |
| `depthWrite` | `false` |
| `depthTest` | `false` |

**作用**：给点阵提供一个微弱的暗绿底色，防止"空心"感。

### 2.2 点阵主体（Dot Matrix，≈8000–12000 实际点）

| 属性 | 值 |
|------|-----|
| 候选点数 | `18000`（经陆地/海洋过滤后约 8000–12000 实际渲染） |
| 点大小 | `3.0`（像素） |
| 亮度倍率 | `2.4` |
| Alpha 提升 | `1.85` |
| 混合模式 | `AdditiveBlending` |
| 深度测试 | `false` |
| 渲染层级 | `renderOrder: 999`（最后渲染） |

**陆地点颜色**：`(0.58, 1.0, 0.62)` — 淡绿色  
**海洋点颜色**：`(0, 0, 0)` — 黑色（仅约 15% 海洋点保留作为极暗点缀）  
**海岸点颜色**：`(0.9, 1.0, 0.72)` — 浅黄绿高光（见 2.3）

**Shader 特性**：自定义 `ShaderMaterial`，实现：
- **闪烁**：`flicker = 0.90 + 0.10 * sin(time * 1.15 + phase)`，每点独立相位
- **朝向衰减**：基于 `dot(worldNormal, viewDirection)` 的面朝因子，背面自动淡出
- **选中的点高亮**：受 `selectedVector` 影响的点变琥珀色并放大

### 2.3 海岸高光层（Coast Dots）

| 属性 | 值 |
|------|-----|
| 采样步长 | `0.9°`（经纬度间隔） |
| 点大小 | `2.8` |
| 亮度倍率 | `3.0` |
| Alpha | `0.92` |
| 球面半径 | `1.024`（略高于普通点的 1.015） |

**作用**：在地球大陆海岸线上产生额外的高亮光点，增强立体感和地图感。用 8 邻域判断法检测海岸像素，约 22% 的海岸点被随机省略（`hash > 0.78` 跳过）。

### 2.4 辉光层（Glow）

| 属性 | 值 |
|------|-----|
| 几何 | `SphereGeometry(1.04, 96, 96)` |
| 材质 | `MeshBasicMaterial`，`side: BackSide` |
| 颜色 | `#9dff9f` |
| 透明度 | `0.05` |

**作用**：从球体背面透出的极淡绿辉光，模拟大气散射。

### 2.5 轨道层（Orbits，2条）

| 属性 | 值 |
|------|-----|
| 形状 | `EllipseCurve(0, 0, 1.32, 1.32, -1.15, 1.15)`，96 段 |
| 线条颜色 | `#57f0d0`（青绿） |
| 线条透明度 | `0.012` / `0.01` |
| 旋转角度 | `orbit1: (0.8, 0.2, 0.4)`，`orbit2: (-0.45, 0.7, -0.2)` |
| 自转速度 | `0.00055 + index * 0.00014` |

每条轨道附带 5 个微型卫星（`SphereGeometry(0.006)`，透明度 0.08），沿轨道缓慢移动。

**作用**：增加科技感和空间深度。

### 2.6 任务标记（Mission Markers，所有任务预置）

| 属性 | 值 |
|------|-----|
| 形状 | `SphereGeometry(0.018, 16, 16)` |
| 默认颜色 | `#9dff9f` |
| 默认透明度 | `0.45` |
| 位置 | 球面半径 `1.08` |
| 可见性 | 默认隐藏，选中章节后显现 |
| 朝向衰减 | 同点阵的面朝因子 |

### 2.7 活跃标记（Active Marker）

| 属性 | 值 |
|------|-----|
| 形状 | `SphereGeometry(0.0396, 24, 24)`（2.2 倍） |
| 颜色 | `#ffc76a`（琥珀色） |
| 位置 | 选中任务时出现在半径 `1.1` 处 |

### 2.8 扫描线 / 雷达层

**官网 CSS 中 `.globe-stage::after` 有一条注释掉的扫描线动画**（`mapScan`），但在实际的 Three.js 渲染中**不存在扫描线层**。扫描线仅用于战术地图（`tactical-map`），未用于数字地球。

> ✅ **结论**：数字地球没有扫描线/雷达层。

## 三、光点生成逻辑

### 3.1 点采样算法

使用**黄金角螺旋**（Golden Angle Spiral）在球面上均匀分布候选点：

```
goldenAngle = π * (3 - √5)
for i in 0..17999:
    y = 1 - (i / 17999) * 2
    radiusAtY = √(1 - y²)
    theta = i * goldenAngle
    x = cos(theta) * radiusAtY
    z = sin(theta) * radiusAtY
```

这种方法比经纬度网格更均匀，不会在两极聚集。

### 3.2 陆地/海洋/海岸区分

通过**地球遮罩纹理** `assets/textures/earth-mask.png` 判断：

1. 加载 PNG 到 Canvas，读取像素数据
2. `sampleEarthMask(lat, lon)`：将经纬度映射到纹理像素，取红色通道亮度
   - `brightness > 0.5` → 陆地
   - `brightness ≤ 0.5` → 海洋
3. `isCoastPoint()`：检查当前点 + 8 邻域（±0.8°），若邻域中有不同于当前点的像素 → 海岸

**海洋点过滤**：非陆地 + `hashPoint(lat, lon) < 0.15` 才保留（约 15%），保留的海洋点颜色设为纯黑 `(0,0,0)`，在 Additive 混合下近乎不可见。

### 3.3 点数量

| 类型 | 数量 |
|------|------|
| 候选总数 | 18,000 |
| 陆地 + 保留海洋 | ≈ 8,000–12,000（取决于遮罩） |
| 海岸点 | 取决于海岸线复杂度，通常数千 |

### 3.4 点大小

| 类型 | 大小（像素） |
|------|-------------|
| 普通点 | 3.0 |
| 海岸点 | 2.8 |
| 高亮（选中任务时） | 放大 1.7 倍 + 额外 1.1 倍 |

### 3.5 点亮度

- 陆地基础亮度：`0.95`
- 海洋基础亮度：`0.55`（但颜色为黑色，实际不可见）
- 最终亮度：`base + hash * (陆地 0.15 / 海洋 0.12)`
- Shader 中 `brightness` uniform：`2.4`
- 海岸 Shader 中 `brightness` uniform：`3.0`

### 3.6 闪烁

**是**，所有点都有闪烁：
```
flicker = 0.90 + 0.10 * sin(time * 1.15 + phase)
```
- 每点独立 `phase`（由 `hashPoint(lat, lon) * 2π` 决定）
- 闪烁幅度：±10%
- 全局时间统一：`performance.now() / 1000`

### 3.7 是否随机

部分随机：
- 海洋点的保留（`hash < 0.15`）是伪随机的
- 海岸点的跳过（`hash > 0.78`）是伪随机的
- 点的相位是伪随机的
- **但采样位置是确定性的**（黄金角 + 固定点数）

### 3.8 是否随时间变化

- **位置不变**——点不移动
- **闪烁变化**——透明度随时间波动
- **自转变化**——整个地球在自动旋转（`isAutoRotating`）
- **选中态变化**——当用户点选章节/任务时，部分点高亮为琥珀色

## 五、Godot 复刻建议

### 5.1 ✅ 必须复刻

| 优先 | 要素 | 原因 |
|------|------|------|
| ★★★ | **暗绿球体底色（`#0d281b`，α≈0.16）** | 给点阵提供载体，防止悬空感 |
| ★★★ | **黄金角螺旋点阵采样** | 官网最核心的视觉特征，均匀分布无极点聚集 |
| ★★★ | **Additive 混合模式** | 点阵发光感的根本来源 |
| ★★★ | **陆地/海洋遮罩过滤** | 官网 `earth-mask.png` 是视觉基础 |
| ★★★ | **朝向面衰减（facing factor）** | 让背面的点淡出，产生球体立体感 |
| ★★☆ | **点的闪烁（±10%，独立相位）** | 官网的重要动态特征 |
| ★★☆ | **淡绿辉光层（BackSide，α≈0.05）** | 官网的大气感来源 |
| ★★☆ | **自动旋转 + 拖拽交互** | 用户互动的核心 |

### 5.2 ⚠️ 可以省略

| 要素 | 原因 |
|------|------|
| **轨道 + 微型卫星** | 官网轨道透明度极低（0.012），视觉效果甚微，后期可加 |
| **任务标记（markers）** | Godot 中暂无章节系统，后期需要时再加 |
| **选中高亮变色（琥珀色）** | 官网为章节系统服务，初期不需要 |
| **扫描线动画** | 官网数字地球本身无扫描线，只在战术地图有 |

### 5.3 ❌ 不建议照搬

| 要素 | 原因 |
|------|------|
| **Three.js 的 ShaderMaterial 写法** | Godot 的 Shader 语法（`shader_type spatial` / `canvas_item`）完全不同，建议重写而非移植 |
| **18000 候选点** | 这个数量是针对 Web 性能权衡的，Godot 中可以使用更多点（30000–50000）且性能更好 |
| **`depthTest: false`** | Three.js 因 WebGL 限制需要这个，Godot 中应使用正确的深度测试以避免排序问题 |
| **点大小 3.0px 像素值** | Godot 是 3D 世界空间，点大小需用世界单位而非像素 |

### 5.4 Shader 分层建议

如果用 Shader 实现，建议分 4 层：

1. **Layer 0 — 球体底色**（`spatial` 材质，`#0d281b`，α=0.16，`blend_mode = mix`）
2. **Layer 1 — 点阵主体**（`spatial` + `particle` 或 `vertex` Shader，Additive 混合，黄金角分布，遮罩过滤）
3. **Layer 2 — 海岸高光**（与 Layer 1 同材质，但额外叠加一组更亮的点）
4. **Layer 3 — 辉光**（`spatial`，BackSide 渲染，`#9dff9f`，α=0.05）

### 5.5 SubViewport + SphereMesh 关键参数

如果用 SubViewport + SphereMesh 将 Web 点阵方案转为 Godot 3D 方案：

| 参数 | 建议值 | 说明 |
|------|--------|------|
| **球体半径** | `1.0` | 单位球方便计算 |
| **经纬线分段** | `96 × 96`（官网值） | 可以降到 64 以节省性能 |
| **SphereMesh 材质** | `unshaded` | 官网使用的是 `MeshBasicMaterial` |
| **SubViewport 大小** | 匹配实际渲染尺寸，建议 `512×512` 以上 | 如果点阵在 Viewport 中预渲染 |
| **点大小（世界单位）** | `0.008–0.015` | 需根据实际视距调试，不要直接用像素值 |

**最关键参数**：
1. 点大小（世界单位）
2. 点亮度（Additive 强度）
3. 朝向衰减范围（`smoothstep(0.02, 0.35, facing)`）

## 六、给 Godot 实现员的下一步调整建议

基于官网效果和现有 Godot 进度（暗绿背景 ✅、右侧球体 ✅、极淡经纬线 ✅、纳米点阵 ✅、黑色遮罩已解决 ✅），建议下一步按此顺序调整：

### 1️⃣ 点阵密度

| 当前状态 | 建议 |
|----------|------|
| 未知（纳米点阵） | 官网约 8000–12000 实际点。**建议 Godot 中增加到 20000–30000**，因为 3D 场景下点的视觉密度会显得更低。使用黄金角螺旋算法。 |

### 2️⃣ 点阵亮度

| 当前状态 | 建议 |
|----------|------|
| 未知 | 官网 Shader `brightness` = 2.4。**建议将点亮度提升到 2.0–3.0 范围**，配合 Additive 混合，使点有"自发光"感。如果亮度不够，背景会显得灰暗。 |

### 3️⃣ 点大小

| 当前状态 | 建议 |
|----------|------|
| 未知 | 官网像素 3.0。**在 3D 世界空间中，建议从 0.01 开始调试**，原则是：远景能看到点阵纹理，近景不糊成一片。不同距离可能需要 LOD。 |

### 4️⃣ 边缘光

| 当前状态 | 建议 |
|----------|------|
| 未实现 | 官网使用背面辉光层（`BackSide`，`#9dff9f`，α=0.05）。**建议添加**：在球体背面渲染一个略大的半透明发光球，颜色 `#9dff9f`，透明度 0.03–0.08。能显著提升立体感。 |

### 5️⃣ 球体透明度

| 当前状态 | 建议 |
|----------|------|
| 已有暗绿背景 | 官网球体透明度 0.16。**建议设置为 0.12–0.20**，让点阵背后有一个极淡的暗绿底色，但又隐约透出背景。过于透明会失去"球体感"，过于不透明会遮挡背景。 |

### 6️⃣ 背景光源范围

| 当前状态 | 建议 |
|----------|------|
| 暗绿背景均匀 | 官网背景是 `radial-gradient(circle at top left, #183f2b, #080c0a 35%, #030403 100%)`。**建议在背景左上角增加一个淡绿色光源径向渐变**，半径约为屏幕高度的 60%，产生"左上打光"的感觉。 |

### 7️⃣ 是否需要海岸高光 / 不规则亮点

**强烈建议添加！** 这是官网视觉中最精致的细节之一：

- **海岸高光**：用遮罩检测陆地边缘，在海岸线上叠加更亮的点（颜色 `#e6ffb8`，亮度 3.0）
- **不规则亮点**：官网通过 `hashPoint()` 为每个点增加 ±12–15% 的随机亮度变化，让点阵看起来自然不呆板
- 实现方式：在遮罩采样时，额外做一次 8 邻域边缘检测

### 8️⃣ 是否需要 orbit / marker / 扫描线

| 要素 | 建议 |
|------|------|
| **Orbit（轨道线）** | **可以后期再加**。官网轨道透明度极低（0.012），视觉效果很微妙。等点阵和光源调好后再加不迟。 |
| **Marker（任务标记）** | **暂不需要**。官网 marker 服务于章节交互系统，Godot 暂时没有对应功能。 |
| **扫描线** | **不需要**。官网数字地球本身没有扫描线。扫描线只出现在战术地图上。 |

## 优先调整顺序建议

```
第一优先级（视觉根基）：
  1. 点阵密度 → 调到 20000–30000，黄金角采样 + 遮罩过滤
  2. 点阵亮度 → 调到 2.0–3.0，Additive 混合
  3. 弹体透明度 → 调到 0.12–0.20，颜色 #0d281b

第二优先级（立体感）：
  4. 边缘辉光 → 背面半透明发光层
  5. 背景光源 → 左上角径向渐变
  
第三优先级（精致度）：
  6. 海岸高光 → 遮罩边缘检测
  7. 不规则亮度变化 → hash 随机化
  8. 点闪烁 → ±10%，独立相位

后期可加：
  9. Orbit 轨道线
  10. 交互拖拽 + 自动旋转
```
